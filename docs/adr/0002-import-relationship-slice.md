# Slice 2 adds repo-local import relationship queries

The second slice of Petrichor extends the Repository Index from definition-only data to repo-local Import Relationships, exposed as two file-targeted Structural Queries: `imports <repositoryPath>` and `importers <repositoryPath>`. This slice stays exact and static by resolving repo-local edges with TypeScript compiler options, including `import`, `export ... from`, type-only, and side-effect relationships, while excluding dynamic `import()`. We chose this over symbol-targeted queries or general references because Repository Paths are exact in the current contract; each query returns exact edge records with metadata, yields `ok` with zero relationships when an indexed file has none, and returns a structured error when the target path is not present in the Repository Index.
