# Slice 3 adds direct caller/callee queries

The third slice of Petrichor extends the Repository Index from symbols plus import edges to direct repo-local Call Relationships between top-level named function declarations with bodies, exposed as two exact, case-sensitive Structural Queries: `callers <functionName>` and `callees <functionName>`. This slice stays narrow and deterministic by using TypeScript symbol resolution to follow aliases, namespace access, and re-exports back to the original indexed function declaration, while excluding methods, function-valued variables, anonymous functions, constructor calls, JSX usage, module-scope calls, callback references, and overload signatures without bodies. We chose exact function-name queries with explicit `subjects` and per-call `caller` / `callee` / `callSite` records over unique symbol IDs or broader usage queries because Petrichor’s current contract is exact and machine-readable, same-name ambiguity must stay visible in the payload, and the goal of this slice is to answer direct “who uses this?” questions without silently widening into a general references engine.
