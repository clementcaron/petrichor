# Slice 5 adds a separate search query

Slice 5 extends Petrichor with a separate `search <query>` Structural Query instead of widening exact `lookup`, because exploratory retrieval is a different contract from Definition Lookup and needs ranked mixed results. The Search Query stays within the current `.ts` / `.tsx` Repository Index, searches both structural fields and indexed file source text, returns a deterministic global top 10 of mixed Symbol-anchored and Repository Path-anchored Search Results, and exposes machine-readable Search Evidence rather than raw relevance scores. We chose this over fuzzy `lookup`, symbol-only search, path-only search, uncapped result dumps, and opaque BM25-first ranking so Petrichor can bridge exact lookup and exploratory navigation without losing its current deterministic, machine-readable CLI shape.
